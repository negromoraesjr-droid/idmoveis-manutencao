📋 Como configurar — 5 passos (gratuito)
1. Acesse console.firebase.google.com
2. Clique em "Criar projeto" → nome: sgcm-idmoveis → Continuar
3. No projeto: menu lateral → "Realtime Database" → Criar banco → Modo teste → Ativar
4. Ainda no projeto: ⚙ → "Configurações do projeto" → "Seus apps" → clique em </> (Web) → Registrar app
5. Copie o bloco firebaseConfig = { ... } e cole no campo abaixo
📌 Regras do Realtime Database (copie e cole no Firebase)
{ "rules": { "sgcm": { ".read": true, ".write": true } } }
No Firebase: Realtime Database → Regras → cole o código acima → Publicar

📖 Instalação, Atualizações e Backup

🚀 Como instalar como app
Android (Chrome):
Abra o arquivo → menu ⋮ → "Adicionar à tela inicial" → Instalar
iPhone (Safari):
Abra o arquivo → botão compartilhar ⬆ → "Adicionar à Tela de Início"
☁ Acesso ilimitado + sync
Com Firebase configurado, o acesso é ilimitado para quantos usuários e dispositivos precisar. Os dados ficam na nuvem Google.

Plano gratuito: até 1GB dados, 50k leituras/dia — suficiente para anos.
Plano pago: a partir de US$25/mês para uso intenso.
💾 Estratégia de Backup
✅ Firebase Realtime DB: backup automático diário pelo Google
✅ Botão "⬇ Backup JSON": exporta tudo em um arquivo. Guarde no Google Drive.
✅ Botão "📋 Exportar CSV": para abrir no Excel.
✅ Recomendação: backup JSON semanal + Firebase ativo = proteção total.
🔄 Como atualizar o app
1. Receba o novo arquivo .html atualizado
2. Antes: exporte backup JSON dos dados atuais
3. Abra o novo arquivo → importe o backup JSON
4. Com Firebase: basta abrir o novo arquivo e configurar o mesmo projeto — os dados sincronizam automaticamente
✅ Com Firebase: zero perda de dados na atualização
💡 Recomendação para a empresa:
Configure o Firebase (5 min, gratuito) → todos os celulares sincronizam em tempo real → dados salvos na nuvem Google → backup automático diário. Faça também um backup JSON semanal no Google Drive como segunda camada de segurança.
